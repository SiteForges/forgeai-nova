# ForgeAI Nova

ForgeAI Nova is a dark, account-based AI chat app built with HTML, CSS, JavaScript, Node.js, Express, Groq, and Ollama fallback.

## What it includes

- Clean black-and-white chat workspace with a simple blue ForgeAI logo
- Local account sign-in with name and email
- Guest mode for this device
- Account-isolated chat history in LocalStorage
- Settings workspace with dedicated sections for:
  - Account
  - AI controls
  - API keys
  - Memory
  - Billing
- Groq-first responses with Ollama local fallback
- Optional Tavily live search for current answers
- Per-account memory for saved profile facts and notes
- Token usage tracking with free and Pro limits
- Copy and regenerate buttons on assistant messages

## Project structure

- `server.js` - backend routes, provider logic, account usage, memory, and API keys
- `public/index.html` - app markup
- `public/style.css` - UI styling and responsive layout
- `public/app.js` - frontend state, LocalStorage, UI, and API calls
- `forgeai-accounts.json` - local account and usage store generated at runtime
- `.env.example` - environment template

## Environment variables

Create `.env` from `.env.example` and add:

```env
GROQ_API_KEY=your_groq_api_key_here
TAVILY_API_KEY=your_tavily_api_key_here_optional
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_MODEL=qwen3:4b
GOOGLE_CLIENT_ID=
OWNER_EMAILS=thisisalexanderbatti@gmail.com
OWNER_ADMIN_TOKEN=
```

Notes:

- `GROQ_API_KEY` powers the main cloud provider.
- `TAVILY_API_KEY` is optional and enables live web search.
- `OLLAMA_BASE_URL` and `OLLAMA_MODEL` are used for local fallback mode.
- `GOOGLE_CLIENT_ID` is not required for the current UI.
- `.env` stays local and is ignored by Git.


```powershell

```

Then open:

`http://localhost:3000`

## Account behavior

- Local accounts are tied to the email entered at sign-in.
- Chats are stored per account on the current device.
- Guest mode uses a device-local guest account.
- Signing out does not delete that account's saved local chats unless you clear them.

## Usage and plans

- Free plan: `200` tokens, renewed every 2 days
- Pro plan: `1000` tokens, renewed every 2 days after manual verification
- Owner accounts can be unlimited when configured on the backend

## Memory

Nova can remember:

- Name
- Job
- Location
- Custom memory notes

Memory can be saved in two ways:

1. In the Memory section of Settings
2. Naturally through chat messages like `remember that I prefer concise answers`

Memory is stored per account in the local backend store.

## AI provider behavior

Provider order in Auto mode:

1. Groq
2. Ollama fallback

Other modes:

- `Groq only`
- `Ollama only`

## Ollama fallback setup

1. Install Ollama
2. Start Ollama
3. Pull the local model:

```powershell
ollama pull qwen3:4b
```

4. Start ForgeAI Nova:

```powershell
npm start
```

If Groq fails, Auto mode will use Ollama.

Important:

- Ollama is local and does not need an API key
- Ollama does not have live web data unless Tavily is connected
- Bigger models need more RAM and a stronger machine

## Tavily live search

When Web mode is on, ForgeAI Nova can search live web data first for topics like:

- News
- Weather
- Prices
- Sports
- Current events

If `TAVILY_API_KEY` is missing, the app still works and explains that live web search is not set up yet.

## API keys

ForgeAI Nova can create account-linked API keys through the Settings -> API section.

- Keys are generated by the backend
- Keys are hashed before storage
- API usage is deducted from the same account balance
- API route:

`POST /api/v1/chat`

## Common issues

### Groq unavailable

- Check `GROQ_API_KEY`
- Try Auto mode or Ollama only mode

### Ollama unavailable

- Make sure Ollama is running
- Check `OLLAMA_BASE_URL`
- Make sure the model exists:

```powershell
ollama list
```

### Live web search unavailable

- Add `TAVILY_API_KEY`
- Turn on Web mode in Settings -> AI

### Chats mixing between users

Chats are now stored under account-specific LocalStorage keys. If older browser data is still confusing the UI, clear local site storage once and sign back in.

### Port 3000 already in use

Stop the existing process or run with a different port:

```powershell
$env:PORT=3001
npm start
```

## Deployment

Deploy on any Node.js host such as Render, Railway, Fly.io, or a VPS.

Deployment checklist:

- Add `GROQ_API_KEY`
- Add `TAVILY_API_KEY` if needed
- Add `OLLAMA_*` only if your deployment also has local Ollama access
- Run `npm install`
- Start with `npm start`
